# SecuClaw 验收标准 (Definition of Done)

## 1. 功能验收

### 1.1 后端API
每个API必须满足：
- [ ] **API响应时间** < 200ms (无数据库查询)
- [ ] **API响应时间** < 2s (有数据库查询)
- [ ] **错误处理** - 所有API必须有错误处理，返回有意义的错误信息
- [ ] **参数验证** - 所有参数必须验证类型和必需性

### 1.2 前端页面
每个页面必须满足：
- [ ] **页面加载时间** < 3s (首屏)
- [ ] **数据加载** - 加载时显示loading状态
- [ ] **错误展示** - API失败时显示友好的错误信息
- [ ] **空状态** - 无数据时显示空状态提示

### 1.3 功能模块

| 模块 | 验收条件 |
|-----|---------|
| 仪表盘 | 显示所有6个域的KPI指标，数据实时更新 |
| 作战室 | P1/P2事件水平排列，团队状态正确显示 |
| 安全事件 | 列表/详情/创建/状态变更完整流程 |
| 漏洞管理 | CVE列表/详情/状态更新完整流程 |
| 威胁情报 | APT/勒索/钓鱼等类型正确分类 |
| 合规审计 | GDPR/SOC2/ISO27001等框架合规状态 |
| 资产管理 | 资产发现/分类/漏洞关联完整 |
| 知识库 | MITRE ATT&CK战术/技术可搜索 |
| AI专家 | 8种角色可切换，响应正常 |

## 2. 技术验收

### 2.1 代码质量
- [ ] **TypeScript编译** - 无error (warning可接受)
- [ ] **单元测试** - 核心业务逻辑有单元测试
- [ ] **代码规范** - ESLint/Prettier通过

### 2.2 性能
- [ ] **首屏加载** < 3s
- [ ] **API响应** < 200ms (无查询) / < 2s (有查询)
- [ ] **内存占用** < 500MB (前端)

### 2.3 安全
- [ ] **输入验证** - 所有用户输入验证
- [ ] **XSS防护** - 无存储型XSS
- [ ] **API认证** - 敏感API需认证
