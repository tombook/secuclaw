# SecuClaw 回滚计划

## 1. 目的

定义系统故障时的回滚策略，确保：
- 最小化服务中断时间
- 保护数据完整性
- 快速恢复服务
- 减少业务影响

## 2. 回滚触发条件

### 2.1 必须回滚 (P0)
| 条件 | 说明 |
|-----|------|
| 数据损坏/丢失 | 数据库损坏、数据被误删 |
| 安全漏洞 | 发现严重安全漏洞需要立即修复 |
| 系统不可用 | 服务完全无法访问 |
| 核心功能失效 | 主要业务功能无法使用 |

### 2.2 建议回滚 (P1)
| 条件 | 说明 |
|-----|------|
| 性能严重下降 | 响应时间增加 > 5倍 |
| 频繁错误 | 错误率 > 10% |
| 非核心功能失效 | 但影响用户体验 |

### 2.3 可选回滚 (P2)
| 条件 | 说明 |
|-----|------|
| UI显示异常 | 不影响功能 |
| 非关键功能失效 | 用户可绕过 |

## 3. 回滚等级

### 3.1 等级定义

| 等级 | 影响范围 | 回滚时间 | 决策人 |
|-----|---------|---------|--------|
| L0 | 全部系统 | 30分钟内 | Tech Lead + CEO |
| L1 | 单个服务 | 15分钟内 | Tech Lead |
| L2 | 单个功能 | 5分钟内 | 开发负责人 |

### 3.2 回滚决策矩阵

```
                    影响范围
                小        中        大
紧急程度  高    L2         L1        L0
          中    L2         L1        L1
          低    不回滚     L2        L1
```

## 4. 回滚流程

### 4.1 紧急回滚流程 (< 15分钟)

```
┌─────────────────────────────────────┐
│  1. 发现问题 (监控/用户报告)        │
└─────────────────┬───────────────────┘
                  ↓
┌─────────────────────────────────────┐
│  2. 确认问题严重性                   │
│     - 检查监控系统                    │
│     - 确认影响范围                    │
│     - 评估是否需要回滚               │
└─────────────────┬───────────────────┘
                  ↓
┌─────────────────────────────────────┐
│  3. 触发回滚 (如果需要)              │
│     - 通知团队                       │
│     - 确定回滚等级                   │
│     - 准备回滚                       │
└─────────────────┬───────────────────┘
                  ↓
┌─────────────────────────────────────┐
│  4. 执行回滚                         │
│     - 停止新部署                     │
│     - 执行数据库回滚 (如需要)         │
│     - 恢复上一版本                   │
│     - 验证服务状态                   │
└─────────────────┬───────────────────┘
                  ↓
┌─────────────────────────────────────┐
│  5. 通知相关方                      │
│     - 通知用户群                     │
│     - 更新状态页面                   │
│     - 发送事故报告                   │
└─────────────────┬───────────────────┘
                  ↓
┌─────────────────────────────────────┐
│  6. 事后分析                         │
│     - 分析根因                       │
│     - 制定修复计划                    │
│     - 更新部署流程                    │
└─────────────────────────────────────┘
```

### 4.2 回滚检查清单

#### 回滚前
```
□ 确认问题复现
□ 通知团队成员
□ 记录当前时间点
□ 备份当前数据库状态 (如需要)
□ 准备回滚命令
□ 确认回滚执行人
□ 通知用户可能的服务中断
```

#### 回滚执行中
```
□ 停止部署流程
□ 执行数据库回滚 (如需要)
□ 切换到上一版本
□ 验证服务启动
□ 检查核心功能
□ 确认数据完整性
```

#### 回滚后
```
□ 通知团队回滚完成
□ 通知用户服务恢复
□ 更新状态页面
□ 创建事故记录
□ 开始根因分析
```

## 5. 版本管理

### 5.1 版本命名

```
格式: vYY.MM.DD.HHmm
示例: v26.03.26.0930
```

### 5.2 版本记录

每次部署记录：
```
## v26.03.26.0930
日期: 2026-03-26 09:30
部署人: @XX
内容:
- 功能A
- 修复B
- 优化C
回滚版本: v26.03.25.1800
```

### 5.3 版本存储

| 类型 | 位置 | 保留时间 |
|-----|------|---------|
| 前端构建 | /dist/vXX.XX.XX.XX | 10个版本 |
| 后端镜像 | /dist/vXX.XX.XX.XX | 10个版本 |
| 数据库备份 | /backup/ | 7天 |
| 配置备份 | /config/backup/ | 30天 |

## 6. 数据库回滚

### 6.1 自动备份

| 触发时机 | 备份内容 | 保留时间 |
|---------|---------|---------|
| 每次部署前 | 全量数据库 | 24小时 |
| 每日凌晨 | 全量数据库 | 7天 |
| 每小时 | 增量日志 | 24小时 |

### 6.2 数据库回滚命令

```bash
# 停止应用
pm2 stop secuclaw

# 恢复数据库 (如需要)
pg_restore -h localhost -U secuclaw -d secuclaw_db /backup/pre-deploy-20260326.sql

# 切换到上一版本
cd /opt/secuclaw
git checkout v26.03.25.1800

# 重启应用
pm2 start secuclaw
```

## 7. 前端回滚

### 7.1 前端回滚命令

```bash
# 切换到上一版本
cd /opt/secuclaw/ui
git checkout v26.03.25.1800

# 重新构建
npm run build

# 重启前端服务
pm2 restart secuclaw-ui
```

### 7.2 快速回滚脚本

```bash
#!/bin/bash
# rollback.sh - 快速回滚脚本

VERSION=$1
if [ -z "$VERSION" ]; then
    echo "Usage: ./rollback.sh <version>"
    echo "Available versions:"
    ls -la /opt/secuclaw/dist/
    exit 1
fi

echo "Rolling back to $VERSION..."

# 停止服务
pm2 stop secuclaw secuclaw-ui

# 恢复代码
cd /opt/secuclaw
git checkout $VERSION

# 重启服务
pm2 start secuclaw secuclaw-ui

echo "Rollback complete!"
```

## 10. 复盘会议

### 10.1 复盘时间
- P0事故: 24小时内
- P1事故: 48小时内
- P2事故: 下周站会

### 10.2 复盘议程

```
1. 事故回顾 (15分钟)
   - 时间线
   - 影响范围
   - 处置过程

2. 根因分析 (30分钟)
   - 直接原因
   - 根本原因
   - 遗漏的预警信号

3. 改进措施 (30分钟)
   - 短期措施 (1周内)
   - 中期措施 (1月内)
   - 长期措施 (季度)

4. 行动项跟踪
   - 指定负责人
   - 确定完成时间
```
