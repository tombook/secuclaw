
# OpenSpec 提案: 角色仪表盘UI精简+能力对齐优化
## 版本: v1.0
## 日期: 2026-04-19
## 状态: 草稿 → 待审批 → 已执行

---
### 问题定义
1. **UI冗余问题**: 多个角色仪表盘存在重复指标展示（如CISO面板'44/100'出现4次），信息密度低
2. **结构空白问题**: '状态栏'和'工具事件汇总'之间区域设计未对齐预期，存在无效空白
3. **能力对齐问题**: 面板内容未完全与`security/skills/`下的角色light安全能力1:1映射

### 优化目标
✅ **UI减法**: 每个核心工具面板仅保留1个核心大指标，移除所有重复展示内容
✅ **分层结构**: 状态栏 → 5核心工具面板 → 工具事件汇总 三层结构100%填充无空白
✅ **能力对齐**: 所有面板数据/图表完全对应角色light安全能力定义，无无关内容
✅ **全角色覆盖**: 优化应用到全部8个角色，确保设计一致性

### 范围
- 仅修改 `src/components/role-commander/sc-role-commander.ts` 文件
- 不影响现有API、工具执行、权限逻辑
- 兼容现有浏览器访问，无功能破坏性变更
