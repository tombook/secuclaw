# Proposal: 角色仪表盘主面板内容升级

## 意图

每个安全角色的"状态栏"和"工具事件汇总"之间，需要展示**基于角色安全能力的动态监控面板**。当前各角色的专属面板（`_renderRoleDashboard()`）内容偏简单，需要深度对齐各角色的 SKILL.md 安全能力。

## 范围

涉及 8 个角色的主面板内容：

| 角色 | 安全能力 (SKILL.md) | 当前面板 | 需要升级为 |
|------|-------------------|---------|-----------|
| CISO | SEC+LEG+IT 三元 | 待审批+治理记录+报告 | 战略决策看板：风险趋势+合规雷达+预算热力图 |
| 指挥官 | SEC+LEG+IT+BIZ 四元 | 活跃事件+角色协同 | 全域态势：事件分级+AI调度+跨部门协调流 |
| 安全运营 | SEC+IT+BIZ 三元 | 告警+SOAR | SOC运营：告警趋势+威胁狩猎+值班日历 |
| 安全专家 | SEC+IT 二元 | CVE列表 | 漏洞管理：CVE时间线+补丁SLA+攻击面变化 |
| 隐私官 | SEC+LEG 二元 | 合规进度+数据流 | 隐私治理：DPIA进度+数据主体请求+跨境审计 |
| 架构师 | SEC+IT 二元 | STRIDE+变更 | 架构治理：零信任路线图+信任边界图+技术债 |
| 业务安全 | SEC+BIZ 二元 | BCP状态 | 业务韧性：BCP/DR矩阵+业务影响热图+ROI |
| 供应链 | SEC+LEG+BIZ 三元 | 供应商评级 | 供应链图谱：风险评估矩阵+合同到期+SBOM热图 |

## 设计原则

1. **数据驱动** — 面板内容来自 mock 数据（未来接 API）
2. **角色能力对齐** — 每个面板对应 SKILL.md 的 light 能力
3. **可视化优先** — 用进度条、状态条、小型图表而非纯文字
4. **可交互** — 点击项目可跳转到对应工具面板
5. **响应角色切换** — 切换角色时自动更新

## 方案

为每个角色增加 2-3 个监控面板（panel），取代当前的简单列表。使用 CSS Grid/Flex 排列。

### CISO 升级示例
```
┌──────────────────────────┬──────────────────────────┐
│ 📊 风险趋势（30天）       │ 🎯 合规雷达（4法规）      │
│ 折线图：风险评分走势       │ 雷达图：GDPR/PIPL/网安/CCPA│
│ + 关键事件标注            │ + 差距数量                │
├──────────────────────────┼──────────────────────────┤
│ 💰 预算热力图             │ 📋 治理审批流              │
│ 按类目×季度 热力显示      │ 策略→评审→审批 状态管道    │
└──────────────────────────┴──────────────────────────┘
```

## 技术方案

- 修改 `_renderCisoDashboard()` 等 8 个方法
- 新增小型 SVG 图表（趋势线、雷达图、热力图）
- CSS Grid 2×2 或 3×2 布局
- 数据源：扩展 `ROLE_DASHBOARDS[roleId]` 配置
