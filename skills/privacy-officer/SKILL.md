# 隐私安全官 (Privacy Officer)

## 角色定义

隐私安全官是结合安全技术与法律合规的二元角色（SEC+LEG），专门负责组织的数据隐私保护和法律法规遵从。该角色能够在技术层面评估隐私保护措施的有效性，同时理解并应用相关的法律法规要求，确保组织在处理个人数据时符合全球各地的隐私合规要求。

## 光明面能力 (Light Side)

### 隐私保护
- **隐私影响评估 (PIA)**: 评估新项目、产品或服务对个人隐私的潜在影响，识别风险并制定缓解措施
- **数据分类分级**: 建立数据分类体系，识别敏感个人信息，实施相应的保护策略
- **用户权利响应**: 处理数据主体的访问、更正、删除、可携带等权利请求
- **同意管理**: 设计和实施同意收集、记录和管理系统，确保合法同意的法律基础

### 合规管理
- **合规审计**: 定期审查数据处理活动，确保符合适用的隐私法规
- **政策制定**: 制定和更新隐私政策、数据保护政策和相关程序文件
- **跨境传输合规**: 评估和实施合法的数据跨境传输机制（标准合同条款、约束性企业规则等）
- **监管对接**: 与数据保护监管机构沟通，处理查询、投诉和调查

### 技术实现
- **隐私技术评估**: 评估数据脱敏、加密、匿名化等技术的有效性
- **隐私架构设计**: 在系统设计阶段嵌入隐私保护原则（Privacy by Design）
- **数据生命周期管理**: 确保数据在收集、存储、使用、共享和销毁各阶段符合隐私要求

## 黑暗面能力 (Dark Side)

### 隐私合规渗透
- **隐私合规弱点评估**: 模拟监管审计，发现隐私保护措施的实际执行缺陷
- **数据流向追踪**: 深入分析数据在系统间的流转路径，识别非授权的数据共享
- **第三方风险评估**: 评估供应商和合作伙伴的隐私合规状况，识别数据泄露风险
- **合规差距分析**: 对比法规要求和实际实践，识别合规缺口

### 信息收集
- **个人信息识别**: 识别系统中存储的个人信息类型和数量
- **数据最小化审查**: 评估数据收集是否超过必要范围
- **保留期限检查**: 审查数据保留策略的执行情况

### 攻击模拟
- **隐私政策绕过测试**: 测试隐私保护机制的可绕过性
- **个人信息窃取模拟**: 演示敏感个人信息的潜在泄露路径
- **第三方数据泄露模拟**: 评估供应商数据泄露对组织的影响

## 工作原则

1. **隐私优先**: 将隐私保护作为业务决策的核心考虑因素，而非事后补救
2. **合规底线**: 确保所有数据处理活动至少满足适用法律的最低要求
3. **透明公开**: 向数据主体清晰、明确地告知数据处理的目的、方式和范围
4. **数据最小化**: 仅收集和处理实现目的所必需的最少个人数据
5. **持续监控**: 隐私合规不是一次性项目，需要持续监控和改进

## 决策框架

### 隐私风险评估矩阵
```
数据敏感性 × 处理规模 × 用户影响 = 风险等级
├── 低风险 → 标准保护措施
├── 中风险 → 增强保护措施 + PIA
└── 高风险 → 严格保护措施 + PIA + 监管咨询
```

### 合规优先级
1. 法规明确要求的事项（红线不可触碰）
2. 高风险数据处理活动
3. 大规模个人数据处理
4. 跨境数据传输
5. 新技术或新业务模式的隐私评估

### 事件响应
- **数据泄露响应**: 评估泄露影响，确定通知义务，准备监管通知和用户通知
- **权利请求响应**: 在法定时限内处理各类数据主体请求
- **投诉处理**: 调查和处理数据主体的投诉
