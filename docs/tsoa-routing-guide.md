Tsoa Swagger Integration and Routing Guide
Overview:
- This document describes how to configure tsoa to automatically generate Swagger/OpenAPI documentation and to wire up routes in an Express (or similar) server.
- It covers configuration, server integration, and code generation steps.

Assumptions:
- Node.js + TypeScript project
- Express-like application
- You plan to generate Swagger (swagger.json) via tsoa and serve docs at /docs

1) Configure tsoa (root: tsoa.json)
- entryFile: path to your main server file (e.g., ./src/server.ts)
- controllerPathGlobs: pattern pointing to your controllers (e.g., ./src/controllers/**/*Controller.ts)
- spec.outputDirectory: where swagger.json will be generated (e.g., ./swagger)
- routes.routesDir: directory where generated route files will live (e.g., ./src/routes)

2) Wire routes and docs in your server
- In your main server file (e.g., src/server.ts):

  import express from 'express';
  import { RegisterRoutes } from './routes'; // generated by tsoa
  import swaggerUi from 'swagger-ui-express';
  import path from 'path';

  const app = express();

  app.use(express.json());

  // Register all API routes generated by tsoa
  RegisterRoutes(app);

  // Serve Swagger UI at /docs and point it to the generated swagger.json
  const swaggerDir = path.resolve(process.cwd(), 'swagger');
  app.use('/docs', swaggerUi.serve, swaggerUi.setup(undefined, { swaggerUrl: '/docs/swagger.json' }));
  app.get('/docs/swagger.json', (req, res) => {
    res.sendFile(path.join(swaggerDir, 'swagger.json'));
  });

  export default app;

3) Generate code and docs
- Install dev dependencies if not present: tsoa, typescript, @types/express, express, swagger-ui-express
- Generate swagger and routes:
-   npx tsoa spec
-   npx tsoa routes
- Start your app and open http://localhost:<PORT>/docs

4) Common gotchas
- Ensure alphanumeric controller filenames end with Controller.ts and export classes decorated by tsoa (e.g., @Route, @Get, @Post).
- If you use a different server framework, adapt the middleware setup for docs accordingly.

5) Example controller snippet (TypeScript)

  // src/controllers/pingController.ts
  import { Get, Route, Response, SuccessResponse } from 'tsoa';
  @Route("ping")
  export class PingController {
    @Get()
    public async getPing(): Promise<string> {
      return 'pong';
    }
  }

Notes:
- tsoa will scan these controllers and emit a swagger.json describing the API surface.
- The generated routes file will be placed in src/routes and should be registered with your app as shown above.
