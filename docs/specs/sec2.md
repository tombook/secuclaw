🛡️ SecuClaw 12导航菜单专业设计方案
设计方法论：SMART安全
原则	含义	设计体现
Specific	明确目标	每个页面顶部显示"这个页面解决什么问题"
Measurable	可衡量	所有指标都有数值、百分比、趋势箭头
Actionable	可行动	明确的"下一步"按钮，不是只看数据
Relevant	业务相关	技术指标→业务影响（时间/金钱/风险）
Timeline	时间维度	所有数据都有时间范围，SLA倒计时

1️⃣ 资产库 (ASSET)
名称: AssetRepository
职责: 管理企业所有IT/OT/云资产
数据模型:
  Asset:
    id: string (UUID)
    name: string                    # 资产名称
    type: enum                      # server|network|application|database|container|cloud|endpoint|iot
    category: enum                  # production|development|staging|testing
    environment: enum               # on-premise|cloud|hybrid
    
    # 位置信息
    location:
      datacenter: string
      region: string
      zone: string
    
    # 网络信息
    network:
      ip: string
      mac: string
      hostname: string
      ports: number[]
      fqdn: string
    
    # 技术栈
    technology:
      os: string
      osVersion: string
      software: Software[]
      services: Service[]
    
    # 业务属性
    business:
      owner: string                 # 资产负责人
      department: string            # 所属部门
      criticality: enum             # critical|high|medium|low
      dataClassification: enum      # public|internal|confidential|restricted
      businessImpact: number        # 1-10
    
    # 安全属性
    security:
      attackSurface: number         # 攻击面评分
      exposureLevel: enum           # internet-facing|internal|isolated
      lastScanDate: timestamp
      vulnerabilities: VulnRef[]
      compliance: ComplianceRef[]
    
    # 状态
    status:
      health: enum                  # healthy|degraded|offline|unknown
      lastSeen: timestamp
      lifecycle: enum               # active|deprecated|retired
索引:
  - by_ip: [network.ip]
  - by_hostname: [network.hostname]
  - by_owner: [business.owner]
  - by_type: [type, category]
  - by_criticality: [business.criticality]
数据来源:
  - 自动发现: 网络扫描、云API、CMDB同步
  - 手动录入: 资产登记表单
  - 外部集成: ServiceNow, CMDB, 资产管理工具

2️⃣ 漏洞库 (VULN)
名称: VulnerabilityRepository
职责: 管理所有漏洞信息、修复状态、风险评分
数据模型:
  Vulnerability:
    id: string
    cveId: string                   # CVE编号
    
    # 漏洞信息
    info:
      title: string
      description: string
      cvss:
        score: number               # 0-10
        vector: string              # CVSS向量
        severity: enum              # critical|high|medium|low|info
      cwe: string[]                 # CWE分类
      affectedProducts: Product[]
      exploitAvailable: boolean
      exploitInWild: boolean
    
    # 受影响资产
    affectedAssets:
      - assetId: string
        componentName: string
        componentVersion: string
        fixVersion: string
    
    # 修复信息
    remediation:
      status: enum                  # open|in_progress|fixed|wont_fix|risk_accepted
      priority: number              # AI计算优先级 1-100
      assignedTo: string            # 负责人
      dueDate: timestamp
      slaDeadline: timestamp
      fixAvailable: boolean
      fixSteps: string[]
    
    # 风险评估
    risk:
      baseScore: number             # CVSS基础分
      adjustedScore: number         # 调整后风险分
      businessImpact: number        # 业务影响
      exposureScore: number         # 暴露程度
      threatScore: number           # 威胁情报关联
    
    # 证据
    evidence:
      scanSource: string            # 扫描来源
      firstDetected: timestamp
      lastDetected: timestamp
      proofOfConcept: string        # POC
      screenshot: string
    
    # 审计追踪
    audit:
      createdBy: string
      createdAt: timestamp
      updatedBy: string
      updatedAt: timestamp
      history: AuditLog[]
计算规则:
  priority = (
    cvssScore * 0.3 +
    businessImpact * 0.25 +
    exposureScore * 0.2 +
    threatScore * 0.15 +
    exploitWeight * 0.1
  )

3️⃣ 威胁库 (THREAT)
名称: ThreatRepository
职责: 管理威胁情报、APT组织、攻击模式
数据模型:
  ThreatActor:
    id: string
    name: string
    aliases: string[]
    type: enum                      # nation-state|cybercrime|hacktivist|insider|unknown
    motivation: enum[]              # espionage|financial|sabotage|political|personal
    
    # 能力描述
    capabilities:
      sophistication: enum          # advanced|intermediate|novice
      techniques: string[]          # MITRE技术ID
      tools: string[]
      malware: string[]
    
    # 历史活动
    campaigns: Campaign[]
    targets: string[]               # 行业/地区
    indicators: IOCRef[]            # 关联IOC
    
    # 情报来源
    sources:
      - provider: string
        confidence: number
        lastUpdated: timestamp
  IOC (Indicator of Compromise):
    id: string
    type: enum                      # ip|domain|url|hash|email|certificate
    value: string
    threatActor: string             # 关联威胁组织
    
    # 分类
    classification:
      category: enum                # malware|c2|phishing|malicious|suspicious
      confidence: number            # 0-100
      severity: enum
    
    # 上下文
    context:
      firstSeen: timestamp
      lastSeen: timestamp
      sightings: number             # 出现次数
      relatedIOCs: string[]
    
    # 状态

```typescript
interface Metric {
  id: string;
  name: string;           // 指标名称
  category: 'security' | 'compliance' | 'operations';
  value: number;         // 当前值
  unit: string;          // 单位
  trend: 'up' | 'down' | 'stable';
  trendValue: number;    // 变化值
  threshold: number;     // 阈值
  status: 'green' | 'amber' | 'red';
  lastUpdated: Date;
}
```

#### 威胁库 (ThreatRepository)
```typescript
interface ThreatActor {
  id: string;
  name: string;
  aliases: string[];
  type: 'nation-state' | 'cybercrime' | 'hacktivist';
  mitreTechniques: string[];
  campaigns: Campaign[];
}

interface IOC {
  id: string;
  type: 'ip' | 'domain' | 'url' | 'hash';
  value: string;
  confidence: number;    // 0-100
  severity: 'high' | 'medium' | 'low';
  status: 'active' | 'expired' | 'false-positive';
  threatActor?: string;
  tags: string[];
}
```

#### 事件库 (IncidentRepository)
```typescript
interface Incident {
  id: string;
  ticketId: string;      // INC-2024-001
  title: string;
  severity: 'P0' | 'P1' | 'P2' | 'P3' | 'P4';
  status: 'new' | 'confirmed' | 'analyzing' | 'containing' | 'eradicating' | 'recovering' | 'closed';
  
  timeline: {
    detectedAt: Date;
    acknowledgedAt?: Date;
    containedAt?: Date;
    eradicatedAt?: Date;
    recoveredAt?: Date;
    closedAt?: Date;
  };
  
  sla: {
    responseDeadline: Date;
    resolutionDeadline: Date;
    responseBreached: boolean;
    resolutionBreached: boolean;
```
