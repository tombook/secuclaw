## 一、API接口开发标准化流程
### 适用场景：新增业务接口、扩展现有接口
#### 完整执行步骤：
##### 1. 新建路由文件
在 `/packages/core/src/gateway/routes/` 目录下新建文件，命名格式：`{业务域}-routes.ts`
> 参考示例：`auth-routes.ts`、`compliance-routes.ts`

##### 2. 编写路由注册函数
```typescript
// 引入依赖
import type { RouterDeps } from '../../gateway/router.js';
// 引入对应模块的service
import { MyModuleService } from '../../my-module/service.js';

// 导出注册函数
export function registerMyModuleRoutes(
  handlers: Map<string, (params: Record<string, unknown>) => Promise<unknown>>,
  deps: RouterDeps
): void {
  // 注册接口，命名规范：{业务域}.{操作名}
  handlers.set('mymodule.list', (params) => handleList(params, deps));
  handlers.set('mymodule.create', (params) => handleCreate(params, deps));
  handlers.set('mymodule.update', (params) => handleUpdate(params, deps));
  handlers.set('mymodule.delete', (params) => handleDelete(params, deps));
}

// 实现接口处理函数
async function handleList(params: Record<string, unknown>, deps: RouterDeps) {
  // 1. 参数校验
  const { page = 1, pageSize = 10 } = params;
  // 2. 调用service层逻辑
  const service = new MyModuleService(deps.jsonStore);
  // 3. 返回结果
  return service.list({ page, pageSize });
}
```

##### 3. 注册到主路由
在 `/packages/core/src/gateway/router.ts` 中：
```typescript
// 1. 引入注册函数
import { registerMyModuleRoutes } from './routes/my-module-routes.js';

// 2. 在constructor中调用
constructor(deps: RouterDeps) {
  registerMyModuleRoutes(this.handlers, this.deps);
}
```

##### 4. 验证接口
启动项目后调用：`POST http://localhost:21981/api`
```json
{
  "method": "mymodule.list",
  "params": { "page": 1, "pageSize": 10 }
}
```

## 二、前端页面开发与i18n集成流程
### 适用场景：新增前端页面、扩展现有页面功能
#### 完整执行步骤：
##### 1. 新建页面组件
在 `/ui/src/ui/pages/` 目录下新建Lit组件，命名格式：`sc-{页面名}-page.ts`
> 参考示例：`sc-dashboard.ts`、`sc-compliance-page.ts`

##### 2. 接入i18n能力
```typescript
import { LitElement, html } from 'lit';
import { customElement, state } from 'lit/decorators.js';
import { I18nController } from '../../i18n/lib/lit-controller.js';

@customElement('sc-my-page')
export class ScMyPage extends LitElement {
  private i18n = new I18nController(this);

  render() {
    return html`
      <div class="page-container">
        <h1>${this.i18n.t('mypage.title')}</h1>
        <p>${this.i18n.t('mypage.description')}</p>
        <button>${this.i18n.t('common.save')}</button>
      </div>
    `;
  }
}
```

##### 3. 添加多语言翻译
在 `/ui/src/i18n/locales/` 目录下的三个语言文件中添加对应翻译：
```typescript
// en.ts
export const en = {
  mypage: {
    title: 'My Module Page',
    description: 'This is my custom module page'
  }
}

// zh-CN.ts
export const zhCN = {
  mypage: {
    title: '我的模块页面',
    description: '这是我自定义的模块页面'
  }
}
```

## 三、业务模块扩展字段实现方案
### 核心优势：无需数据库迁移、零成本扩展
#### 执行步骤：
##### 1. 类型定义扩展
在模块的 `types.ts` 或 `repository.ts` 中扩展实体接口：
```typescript
export interface Asset {
  id: string;
  name: string;
  ip: string;
  // 新增扩展字段
  department?: string;
  businessLine?: string;
  owner?: string;
  tags?: string[];
  // 通用扩展字段
  extra?: Record<string, any>;
}
```

##### 2. 数据读写支持
无需修改存储层代码，直接读写扩展字段：
```typescript
async function updateAsset(id: string, data: Partial<Asset>) {
  const asset = await this.repo.getById(id);
  if (data.department) asset.department = data.department;
  if (data.extra) asset.extra = { ...asset.extra, ...data.extra };
  await this.repo.save(asset);
  return asset;
}
```
