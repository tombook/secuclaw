# SecuClaw 每日 Standup 机制

## 1. 目的

- 快速同步团队进展
- 识别阻塞问题并及时解决
- 促进团队协作
- 保持项目透明度

## 2. 时间与形式

| 项目 | 内容 |
|-----|------|
| 时间 | 每日 09:30 - 09:45 (15分钟) |
| 地点 | 飞书群 "SecuClaw 每日站会" |
| 形式 | 文字/语音 |

## 3. 参与人员

| 角色 | 是否必须 | 说明 |
|-----|---------|------|
| 技术负责人 | ✅ 必须 | 主持会议，跟踪进度 |
| 前端工程师 | ✅ 必须 | 同步前端进展 |
| 后端工程师 | ✅ 必须 | 同步后端进展 |
| 安全专家 | ⚠️ 按需 | 安全相关议题时参与 |
| 测试工程师 | ⚠️ 按需 | 测试相关议题时参与 |
| 产品经理 | ⚠️ 按需 | 需求澄清时参与 |

## 4. 会议流程

### 4.1 开场 (1分钟)
```
主持人: "大家早上好！开始今日站会"
```

### 4.2 轮流发言 (每人2分钟)

每人按顺序回答三个问题：

**问题1: 昨天完成了什么？**
```
格式:
✅ [任务名称] - [完成状态]
- 简要说明
- 产出物链接 (PR/文档)
```

**问题2: 今天计划做什么？**
```
格式:
🎯 [任务名称] - [计划状态]
- 预计完成时间
- 需要的配合
```

**问题3: 有什么阻塞问题？**
```
格式:
🚧 [阻塞问题] - [紧急程度: P0/P1/P2]
- 问题描述
- 需要谁的帮助
- 期望解决时间
```

### 4.3 阻塞问题讨论 (2分钟)

对于P0/P1阻塞问题：
```
1. 立即在群里@相关人员
2. 确定解决时间和方案
3. 记录在阻塞清单中
```

### 4.4 散会 (1分钟)
```
主持人: "站会结束，大家加油！"
```

## 6. 站会规则

### 6.1 必须遵守
- ✅ 准时参加，不迟到
- ✅ 提前准备好发言内容
- ✅ 简洁明了，不超过2分钟
- ✅ 及时更新阻塞问题状态

### 6.2 禁止行为
- ❌ 长篇大论，超时
- ❌ 隐瞒阻塞问题
- ❌ 私下解决，不在群里同步
- ❌ 无故缺席

## 7. 站会后跟踪

### 7.1 阻塞问题跟踪

| 状态 | 说明 |
|-----|------|
| 🆕 新增 | 刚提出的阻塞问题 |
| 🔄 处理中 | 正在解决 |
| ✅ 已解决 | 问题已解决 |
| ⚠️ 升级 | 需要更高级别介入 |

### 7.2 每日站会记录

自动汇总到文档：
```
/docs/standup/
├── 2026-03-26.md
├── 2026-03-25.md
└── BLOCKERS.md  (阻塞问题汇总)
```

## 8. 常见问题

### Q: 无法参加站会怎么办？
A: 提前在群里发消息说明，并在会后查看会议记录补齐任务更新

### Q: 阻塞问题需要升级怎么办？
A: 在群里@技术负责人和产品经理，说明紧急程度和影响范围

### Q: 站会超时怎么办？
A: 主持人有权打断，细节问题会后单独讨论

### Q: 任务变化了怎么办？
A: 在群里实时更新，同步给相关人员
